# 凡人篇装备设计

状态：草案 v1

适用范围：

- `凡人篇` 主线地图与支线地图
- 当前阶段 `云来镇 -> 荒野 -> 青竹林 -> 玄铁矿洞 / 断碑遗迹 -> 噬魂兽谷 -> 灵脊岭`
- 当前 `5` 个装备槽位：`weapon / head / body / legs / accessory`

配套基础框架：

- [装备框架设计](./equipment-framework.md)

## 1. 为什么现在的装备不成立

当前装备问题不只是“数量少”，而是整套装备体验还没有和凡人篇剧情绑在一起。

现状问题：

- 当前装备内容位于 `packages/server/data/content/items/装备/*.json`，原先单文件规模远不足以支撑前期多地图、多阶段推进
- 大部分装备还是“换个名字的白板数值”，缺少明确剧情来源、玩法定位和流派分工
- 同一槽位缺少横向选择，玩家常常是“哪个数值大穿哪个”，而不是根据打法选装备
- 装备池没有明显体现凡人篇的核心气质：穷、险、旧、残、捡漏、硬撑、叩门前夜
- 现有价值分布已有错位，例如个别黄阶装备量化价值已经压到或接近玄阶区间，后续继续追加会更难收

因此，凡人篇装备设计的目标不是简单把条目从 `15` 件加到 `30` 件，而是把前期整段剧情真正做成一条“装备也会讲故事”的成长链。

## 2. 设计目标

凡人篇装备应同时满足以下目标：

- 让玩家能从装备名字和来源看出自己正处于哪一段剧情
- 每个阶段、每个槽位至少都有 `2 ~ 3` 件能打出不同体验的可选装备
- 让近战硬拼、轻身游斗、凝气术修、守夜勘探这几条前期流派都能成立
- 让“残器”“旧物”“拼装货”成为凡人篇装备特色，而不是一上来就遍地仙家法宝
- 保持与现有数值预算兼容，避免黄阶、玄阶、地阶过早混成一锅

## 3. 剧情驱动的装备原则

凡人篇不是“拿到神器一路碾过去”的故事，而是“先活下来，再慢慢摸到门槛”。

装备设计必须跟着这条叙事线走：

| 剧情层级 | 对应区域 | 建议境界 / 阶段 | 装备气质 | 装备主流品阶 |
| --- | --- | --- | --- | --- |
| 人间求生 | 云来镇、荒野 | `realmLv 1-6` | 旧货、拼装、脚行、猎户、门丁制式 | 凡阶为主，少量黄阶 |
| 半步入玄 | 青竹林、玄铁矿洞、断碑遗迹 | `realmLv 7-14` | 林路旧线、矿卫旧制、残纹器胚、半成灵具 | 黄阶为主，极少数破格玄阶残器 |
| 叩门前夜 | 噬魂兽谷、灵脊岭 | `realmLv 15-24` | 镇煞、护心、引灵、压邪、叩门试器 | 黄阶上限，少量标准玄阶 |

核心要求：

- 越早期的装备越像人间器具，先讲求“能不能活”
- 越后期的装备越开始出现“灵机、阵纹、引气、镇煞”痕迹，但仍不应直接泛滥成修仙篇法宝
- 凡人篇允许出现少量高品阶残器，但它们必须是“线索型强物”，不能成为主流掉落

## 4. 流派轴与装备分工

凡人篇装备至少要撑起以下四条主流方向：

| 流派 | 核心体验 | 重点属性 | 主要来源地图 |
| --- | --- | --- | --- |
| 血战破势 | 顶着伤害上前硬换，靠破招和护甲站住 | `physAtk`、`maxHp`、`physDef`、`breakPower`、`resolvePower` | 荒野、矿洞、兽谷 |
| 轻身游斗 | 先手、机动、侧切、绕打 | `moveSpeed`、`dodge`、`hit`、`crit` | 云来镇、荒野、青竹林 |
| 凝气术修 | 攒灵、稳输出、读条更顺 | `maxQi`、`spellAtk`、`spellDef`、`qiRegenRate`、`maxQiOutputPerTick` | 遗迹、灵脊岭 |
| 守夜勘探 | 跑图、探索、夜战、生存 | `viewRange`、`hpRegenRate`、`lootRate`、`rareLootRate`、`resolvePower` | 云来镇、遗迹、灵脊岭 |

设计约束：

- 常规装备只服务其中 `1` 条主轴，不做“四不像万金油”
- 同一件装备最多同时承担 `1` 个主定位和 `1` 个副定位
- `weapon` 负责输出倾向，`head / accessory` 更适合做识别度词条，`body` 负责稳态，`legs` 负责机动

## 5. 装备池规模规划

凡人篇建议先把装备池扩到 `60 ~ 72` 件，其中只保留少量高稀有剧情装备。

建议分布：

| 区域 | 建议新增后总量 | 主要获取方式 | 说明 |
| --- | --- | --- | --- |
| 云来镇 | `8` | 任务奖励、店铺交换、容器搜寻 | 让玩家刚进镇就开始换掉最破的起始装 |
| 荒野 | `8` | 普通掉落、流匪精英、路边残货 | 建立早期战斗分支 |
| 青竹林 | `10` | 精英掉落、采药点、隐蔽容器 | 拉开轻身和凝气的差别 |
| 玄铁矿洞 | `10` | 矿卫掉落、矿箱搜寻、旧矿首领 | 产出硬战与破招装备 |
| 断碑遗迹 | `10` | 遗迹守怪、残碑搜寻、支线奖励 | 提供第一批真正有“灵气味”的装备 |
| 噬魂兽谷 | `12` | 精英怪、首领掉落、镇煞任务奖励 | 进入叩门前夜的高压装备层 |
| 灵脊岭 | `12` | 终章掉落、叩门任务、隐藏搜寻点 | 收束凡人篇，并为修仙篇做过渡 |

获取结构建议：

- 普通怪掉落负责“常规件”
- 精英怪掉落负责“流派核心件”
- 任务奖励负责“保底换代件”
- 容器 / 地面物品负责“带故事感的偏门件”
- 终章区域只放少量真正值得记住的命名装备

## 6. 当前结构可直接支持的效果边界

按 [装备框架设计](./equipment-framework.md) 的分层，这里说的是 `P0` 能直接落地的部分。

当前项目装备数据已经能稳定承载：

- `equipStats`
- `equipAttrs`
- 常规数值型描述文本

因此凡人篇装备一期应优先使用这些能力：

- 六维加成：`体魄 / 神识 / 身法 / 根骨 / 悟性 / 气运`
- 主资源：`maxHp / maxQi`
- 攻防：`physAtk / spellAtk / physDef / spellDef`
- 对抗：`hit / dodge / crit / critDamage / breakPower / resolvePower`
- 续航：`qiRegenRate / hpRegenRate / maxQiOutputPerTick`
- 功能：`cooldownSpeed / viewRange / moveSpeed / playerExpRate / techniqueExpRate / lootRate / rareLootRate`

一期设计原则：

- 常规件使用 `2 ~ 3` 条数值即可，不要一件装备塞满所有属性
- 命名稀有件可以到 `3 ~ 4` 条，但必须有明确主轴
- `viewRange / lootRate / techniqueExpRate / rareLootRate` 优先放在 `head / accessory`
- `moveSpeed` 优先放在 `legs`
- `maxQiOutputPerTick` 只放在少量“引灵 / 导气”型装备上

## 7. 后续可扩展的装备被动模板

如果后续要让装备“不只是数值”，不要为每件装备单独写一套新逻辑，而应直接落到 [装备框架设计](./equipment-framework.md) 里的 `effects` 配置。

凡人篇优先可扩以下模板：

| 被动模板 | 适用剧情 | 设计意图 |
| --- | --- | --- |
| 夜行 | 云来镇、荒野、灵脊岭 | 夜间提升视野、命中或搜寻效率，体现宵禁与夜路压迫 |
| 清障 | 断道、林路、矿道 | 与拾取、搜寻、互动耗时相关，体现“能干活的人才能活” |
| 余火 | 玄铁矿洞 | 受击后短暂获得破招、化解或灵力输出速率 |
| 残纹 | 断碑遗迹 | 释放技能后短暂提升法术攻击、冷却速度或神识相关收益 |
| 镇煞 | 噬魂兽谷、灵脊岭 | 对异常、压迫、恐慌场景提供稳定收益 |
| 叩门 | 灵脊岭 | 提升突破前准备能力，例如悟性、神识、引气相关收益 |

落地顺序建议：

- `P0` 先扩装备数量和数值辨识度
- `P1` 再给少量命名装备接入 `on_tick / on_cultivation_tick` 型效果
- `P2` 再扩到 `on_move / on_hit / on_kill` 型效果
- `P3` 才考虑套装、成长装备、装备精炼

文档里的“当前可落地效果”默认直接映射到 `equipStats / equipAttrs`。

## 9. 破格装备控制

凡人篇允许少量破格装备，但要严格控量。

建议规则：

- 破格玄阶残器占凡人篇装备池总量不超过 `10%`
- 破格件只出现在 `断碑遗迹 / 灵脊岭 / 隐线奖励`
- 破格件要么数值偏科，要么成长偏科，不做全面超模件
- 破格件应承担“提前让玩家闻到修仙味”的职责，而不是把同层黄阶全部打废

可用方向：

- 高 `spellAtk` 但防御很普通
- 高 `techniqueExpRate` 但战斗收益一般
- 高 `maxQiOutputPerTick` 但没有额外生存
- 双六维小加成，但缺少直接战斗面板

## 10. 数值与投放约束

凡人篇装备文案再强，也必须回到现有预算上收口。

约束规则：

- 凡阶单件尽量落在 `4 ~ 8` 价值
- 黄阶单件尽量落在 `10 ~ 16` 价值
- 标准玄阶单件尽量落在 `28 ~ 40` 价值
- 破格玄阶残器可压到常规玄阶的 `40% ~ 70%`
- 同阶段常规件优先用 `2` 条强相关词条，不靠堆料抬价值
- `viewRange / 经验增幅 / 掉率` 这类功能词条必须和剧情来源匹配，不能无缘无故满地乱放

特别注意：

- 当前报表里已有黄阶装备量化价值冲到 `21` 的情况，凡人篇重做时要避免继续扩大品阶混乱
- `moveSpeed`、`crit`、`dodge`、`breakPower` 都属于容易“体感超标”的属性，单件预算宁可保守
- `techniqueExpRate`、`playerExpRate`、`rareLootRate` 应只给少量饰品，避免形成全身成长装

## 11. 推荐落地顺序

如果按最小可交付来做，建议分三批推进：

### 第一批：先把装备从“能穿”变成“能选”

- 先补齐 `云来镇 + 荒野 + 青竹林` 的装备池
- 每个槽位至少补出 `2` 条明显不同的路线
- 先只做当前结构能支持的纯数值版本

### 第二批：补中段剧情识别度

- 加上 `玄铁矿洞 + 断碑遗迹` 的命名装备
- 开始让矿卫、残纹、旧器胚这些来源形成鲜明风格
- 同步校正现有装备报表，收品阶预算

### 第三批：做终章过渡件

- 完成 `噬魂兽谷 + 灵脊岭` 的叩门装备
- 只挑 `4 ~ 6` 件做真正能记住的命名核心装备
- 如果装备被动结构准备好了，再给终章件接入模板化被动

## 12. 本文档产出要求

凡人篇后续新增装备时，默认都应回答以下问题：

- 这件装备属于哪张图、哪段剧情、哪类人会穿
- 它解决的是输出、机动、生存、引气还是勘探问题
- 它为什么是这个部位，而不是随便塞到别的槽位
- 它和同阶段同槽位装备相比，差异点在哪里
- 它是否压坏了当前品阶预算

如果这些问题答不出来，这件装备大概率还是“换名白板”，不该进正式内容池。
